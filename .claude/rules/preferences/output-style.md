# 出力フォーマット設定

## ダイジェストの書き方

### フォーマット
```markdown
# 転職ニュースダイジェスト YYYY-MM-DD

## 今日のハイライト
（最も重要なニュース1つを2〜3文で要約）

## 転職市場データ
- （求人倍率・年収データなど数字系の情報）

## ノウハウ・攻略情報
- （書類・面接・エージェント活用などの実用情報）

## トレンド・話題
- （SNSでバズっている話題、注目の体験談など）

## ソラの投稿ネタメモ
- ネタ候補1：（テーマ / 切り口 / ターゲット）
- ネタ候補2：（テーマ / 切り口 / ターゲット）
- ネタ候補3：（テーマ / 切り口 / ターゲット）
```

### 注意点
- 各項目は箇条書きで簡潔に
- データがある場合は必ず数字を入れる
- 情報源のURLを可能な限り記載する
- 前日との差分がわかるように書く

## Threads投稿案の書き方

### フォーマット
```markdown
# Threads投稿案 YYYY-MM-DD

## パターン1：攻略・ノウハウ型
（300〜500文字の投稿案）

## パターン2：エージェント活用型
（300〜500文字の投稿案）

## パターン3：気づき・共感型
（300〜500文字の投稿案）
```

### ルール
- ソラ（@sora_tensyoku）のペルソナで書く
- 一人称は使わない（客観的な攻略情報スタイル）
- 丁寧語ベースでカジュアルに
- エージェントを敵視しない
- 暴露系・批判系は禁止
- データや数字を使って説得力を出す
- アフィリリンクは入れない（投稿案の段階では）
- 300〜500文字程度（Threadsの読みやすい長さ）

## 週次ネタ出し（月曜のみ）

### フォーマット
```markdown
# 週次ネタ出し YYYY-MM-DD

## 今週のnote記事候補

### 候補1
- テーマ：
- ターゲット：20代全般 / 第二新卒 / 初転職
- 種類：攻略・ノウハウ / エージェント活用 / データ分析 / 体験談
- なぜ今書くべきか：
- 切り口：
- 収益化：アフィリ向き / 有料note向き / 認知拡大

### 候補2
（同上）

### 候補3
（同上）
```

## 競合分析（日曜のみ）

### フォーマット
```markdown
# 競合分析 YYYY-MM-DD

## 検索結果の傾向
（上位5件の記事・投稿の傾向をまとめる）

## 伸びている切り口
（どんなテーマ・フォーマットが反応を得ているか）

## ソラの差別化ポイント
（競合と比較してどう差別化できるか）
```
